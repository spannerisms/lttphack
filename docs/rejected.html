REASON: Adding hitboxes would require new sprite character graphics to represent those hitboxes, and there just simply is not enough space in video memory for that. It would also require juggling with the vanilla object allocation system and/or introducing an unacceptable amount of lag into the game. Most hitboxes are not all that interesting anyway. The only hitboxes that might provide useful to enable are the sword hitboxes, and you can read about those in my Explication or FoxLisk's notes.

REASON: The change to SA-1 included a complete overhaul of the backend code.
